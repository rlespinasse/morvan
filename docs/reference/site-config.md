# Référence : Configuration du site

Le site consomme trois types d'artefacts générés automatiquement à partir des données brutes : la configuration JavaScript, les GeoJSON reprojetés et l'index des liens inverses.

## Artefacts générés

| Fichier | Producteur | Consommateur |
|---------|------------|--------------|
| `site/src/generated-config.js` | `scripts/generate_site_config.py` | Import ES module dans `site/src/main.js` |
| `site/public/data/layers/**/*.geojson` | `scripts/reproject.py` | Fetch HTTP côté navigateur |
| `site/public/data/reverse-links.json` | `scripts/generate_reverse_links.py` | Fetch HTTP côté navigateur |

Tous sont régénérés par `just prepare`.

## `site/src/generated-config.js`

Module ES exportant trois constantes. Le fichier commence par un en-tête explicite :

```js
// Auto-generated by scripts/generate_site_config.py — do not edit
```

### `layerGroups`

Tableau ordonné des groupes de catégories, dans l'ordre défini par `CATEGORY_ORDER` du script générateur.

```js
export const layerGroups = [
  {
    id: "administratif",
    group: "Administratif",
    layers: [
      {
        id: "administratif--epci",
        label: "Communautés de Communes du PNR du Morvan",
        file: "data/layers/administratif/epci.geojson",
        active: false
      },
      // ...
    ]
  },
  // ...
];
```

| Champ (groupe) | Type | Description |
|----------------|------|-------------|
| `id` | string | Identifiant de catégorie (ex. `administratif`) |
| `group` | string | Libellé affiché (ex. `Administratif`, `Nature & Environnement`) |
| `layers` | `LayerDef[]` | Couches de la catégorie, triées par `label` |

| Champ (couche) | Type | Description |
|----------------|------|-------------|
| `id` | string | Identifiant stable `<catégorie>--<nom>` (double tiret) |
| `label` | string | Nom affiché (issu de `sources.json#name`) |
| `file` | string | Chemin relatif au `base` du site (ex. `data/layers/administratif/epci.geojson`) |
| `active` | boolean \| absent | Absent = active par défaut ; `false` = inactive au démarrage |

La seule couche active par défaut est `administratif--perimetre-parc` (constante `DEFAULT_ACTIVE` du script).

### `styles`

Objet indexé par `layerId`, associant à chaque couche un style Leaflet-Atlas compatible.

```js
export const styles = {
  "administratif--perimetre-parc": {
    color: "#3d7bd9",
    weight: 2,
    fillOpacity: 0.2,
    opacity: 0.8
  },
  // ...
};
```

Les couleurs sont calculées par `generate_layer_colors()` selon la teinte de base de la catégorie (`CATEGORY_BASE_HUES`) et réparties sur ±30° pour maximiser le contraste entre couches sœurs.

Forme du style selon la géométrie :

| Type géométrique | Clés |
|------------------|------|
| `point` | `color`, `radius: 6`, `weight: 1`, `fillOpacity: 0.7` |
| `line` | `color`, `weight: 3`, `opacity: 0.8` |
| `polygon` | `color`, `weight: 2`, `fillOpacity: 0.2`, `opacity: 0.8` |

### `geometryTypes`

Objet indexé par `layerId`, indiquant le type géométrique détecté.

```js
export const geometryTypes = {
  "administratif--perimetre-parc": "polygon",
  "patrimoine-culture/ecomusee": "point",
  "tourisme-economie/grandes-itinerances": "line"
};
```

Valeurs possibles : `"point"`, `"line"`, `"polygon"` (par défaut en cas de géométrie inconnue).

## `site/public/data/layers/`

Copie reprojetée des fichiers de `data/layers/`, structure de dossiers identique :

```text
site/public/data/layers/
├── administratif/
│   ├── communes.geojson
│   ├── departements.geojson
│   └── ...
├── nature-environnement/
│   └── ...
└── ...
```

Différences avec la source `data/layers/` :

- **Projection** : EPSG:4326 (WGS84) au lieu d'EPSG:2154 (Lambert 93)
- **Coordonnées** : arrondies à 6 décimales (≈ 10 cm de précision)
- **Propriété `crs`** : supprimée (RFC 7946 interdit sa présence dans un GeoJSON)
- **Bloc `_source`** : conservé tel quel

Les fichiers sont servis directement par Vite (pas de traitement supplémentaire côté site).

## `site/public/data/reverse-links.json`

Index des relations spatiales bidirectionnelles entre features.

```json
{
  "administratif--communes": {
    "42": {
      "patrimoine-culture--ecomusee": [
        { "index": 3, "label": "Musée de Saint-Brisson" }
      ]
    }
  }
}
```

Structure :

```text
{ "<layerId>": { "<featureIndex>": { "<otherLayerId>": [ { "index", "label" } ] } } }
```

| Champ | Type | Description |
|-------|------|-------------|
| `layerId` | string | Identifiant de couche `<catégorie>--<nom>` |
| `featureIndex` | string (entier) | Index du feature dans le tableau `features` du GeoJSON |
| `otherLayerId` | string | Autre couche liée spatialement |
| `index` | number | Index du feature lié dans l'autre couche |
| `label` | string | Libellé du feature lié (résolu via `LABEL_PROPS` ou champs standards) |

Contraintes :

- Maximum 50 liens par paire (couche, feature, couche cible) — `MAX_REVERSE_PER_LAYER`
- Les 3 couches forestières sont exclues (`SKIP_LAYERS`) pour éviter un index trop volumineux
- Les couches conteneurs (départements, EPCI, communes, entités paysagères, Natura 2000) sont testées dans cet ordre
